## 算法概述

### 基本思想

协同过滤（Collaborative Filtering，CF）是一种经典的推荐算法，其核心思想是：

- **基于用户的协同过滤**：相似的用户往往有相似的偏好，如果用户A和用户B都喜欢某些物品，那么用户A喜欢的其他物品，用户B也可能喜欢。
- **基于物品的协同过滤**：相似的物品往往被相似的用户喜欢，如果用户喜欢物品A，而物品A和物品B很相似，那么用户也可能喜欢物品B。

### 算法分类

1. **基于用户的协同过滤（User-Based CF）**
   - 找到与目标用户相似的其他用户
   - 推荐这些相似用户喜欢的物品

2. **基于物品的协同过滤（Item-Based CF）**
   - 找到与用户已喜欢物品相似的其他物品
   - 推荐这些相似物品

3. **混合推荐（Hybrid）**
   - 结合用户CF和物品CF的结果
   - 通过加权平均得到最终推荐

## 基于用户的协同过滤（User-Based CF）

### 算法原理

基于用户的协同过滤假设：**相似用户有相似的偏好**。

### 计算步骤

#### 步骤1：收集用户行为数据

```
用户A预约过：教练1、教练2、教练3
用户B预约过：教练2、教练3、教练4
用户C预约过：教练1、教练3、教练5
```

#### 步骤2：计算用户相似度

使用 **Jaccard相似度** 计算两个用户的相似程度：

```
Jaccard相似度 = |A ∩ B| / |A ∪ B|
```

其中：
- `A ∩ B`：用户A和用户B共同预约的教练集合
- `A ∪ B`：用户A和用户B预约过的所有教练集合

**示例计算：**

```
用户A预约：{教练1, 教练2, 教练3}
用户B预约：{教练2, 教练3, 教练4}

交集：{教练2, 教练3} → 2个
并集：{教练1, 教练2, 教练3, 教练4} → 4个

相似度 = 2/4 = 0.5
```

#### 步骤3：找到相似用户

- 计算目标用户与所有其他用户的相似度
- 筛选相似度 > 0.1 的用户（阈值可调）
- 按相似度降序排序，取前50个最相似的用户

#### 步骤4：加权统计推荐教练

对每个相似用户预约过的教练进行加权统计：

```
推荐分数(教练X) = Σ(相似度(用户i) × 权重)
```

其中：
- `用户i`：预约过教练X的相似用户
- `相似度(用户i)`：用户i与目标用户的相似度
- `权重`：默认为1，可根据预约次数、时间等因素调整

**排除规则：**
- 排除目标用户已经预约过的教练
- 只推荐目标用户未预约的教练

#### 步骤5：生成推荐列表

- 按推荐分数降序排序
- 取前N个教练作为推荐结果（默认N=10）

### 算法流程图

```
开始
  ↓
获取目标用户的预约记录
  ↓
获取所有其他用户的预约记录
  ↓
计算目标用户与所有其他用户的相似度
  ↓
筛选相似度 > 0.1 的用户
  ↓
统计相似用户预约过的教练（加权）
  ↓
排除目标用户已预约的教练
  ↓
按推荐分数排序
  ↓
返回TopN推荐结果
  ↓
结束
```

## 基于物品的协同过滤（Item-Based CF）

### 算法原理

基于物品的协同过滤假设：**相似的物品被相似的用户喜欢**。

### 计算步骤

#### 步骤1：收集数据

```
目标用户预约过：{教练A, 教练B}

教练A的学员：{用户1, 用户2, 用户3}
教练B的学员：{用户2, 用户3, 用户4}
教练C的学员：{用户1, 用户3, 用户5}
教练D的学员：{用户2, 用户4, 用户6}
```

#### 步骤2：计算教练相似度

使用 **Jaccard相似度** 计算两个教练的相似程度：

```
Jaccard相似度 = |A ∩ B| / |A ∪ B|
```

其中：
- `A ∩ B`：教练A和教练B的共同学员集合
- `A ∪ B`：教练A和教练B的所有学员集合

**示例计算：**

```
教练A的学员：{用户1, 用户2, 用户3}
教练B的学员：{用户2, 用户3, 用户4}

交集：{用户2, 用户3} → 2个
并集：{用户1, 用户2, 用户3, 用户4} → 4个

相似度 = 2/4 = 0.5
```

#### 步骤3：找到相似教练

对目标用户预约过的每个教练：
- 计算该教练与其他所有教练的相似度
- 筛选相似度 > 0.1 的教练（阈值可调）
- 排除目标用户已预约的教练

#### 步骤4：加权统计

对每个相似教练进行加权统计：

```
推荐分数(教练X) = Σ(相似度(教练i, 教练X))
```

其中：
- `教练i`：目标用户预约过的教练
- `相似度(教练i, 教练X)`：教练i与教练X的相似度

#### 步骤5：生成推荐列表

- 按推荐分数降序排序
- 取前N个教练作为推荐结果

### 算法流程图

```
开始
  ↓
获取目标用户预约过的教练列表
  ↓
获取所有教练的学员集合
  ↓
对每个已预约教练：
  ↓
  计算与其他教练的相似度
  ↓
  筛选相似度 > 0.1 的教练
  ↓
  加权累加推荐分数
  ↓
排除目标用户已预约的教练
  ↓
按推荐分数排序
  ↓
返回TopN推荐结果
  ↓
结束
```

## 混合推荐策略

### 策略说明

混合推荐结合了用户CF和物品CF的优势，通过加权平均得到最终推荐结果。

### 权重分配

```
最终推荐分数 = 用户CF分数 × 0.6 + 物品CF分数 × 0.4
```

**权重说明：**
- **用户CF权重：60%** - 更关注用户群体的相似性
- **物品CF权重：40%** - 更关注物品本身的相似性

### 混合策略的优势

1. **互补性**：用户CF和物品CF各有优势，混合可以互补
2. **稳定性**：单一算法失效时，另一种算法仍能提供推荐
3. **准确性**：综合两种算法的结果，提高推荐准确性

### 实现流程

```
开始
  ↓
调用用户CF算法 → 得到推荐列表A
  ↓
调用物品CF算法 → 得到推荐列表B
  ↓
合并推荐列表：
  - 相同教练：加权平均分数
  - 不同教练：分别加入
  ↓
按最终分数排序
  ↓
返回TopN推荐结果
  ↓
结束
```

## 相似度计算方法

### Jaccard相似度

#### 公式

```
Jaccard(A, B) = |A ∩ B| / |A ∪ B|
```

#### 特点

- **取值范围**：[0, 1]
- **值越大**：相似度越高
- **值为0**：两个集合无交集（完全不相似）
- **值为1**：两个集合完全相同（完全相似）

#### 适用场景

- 适合处理**集合数据**（如用户预约的教练集合）
- 不需要评分数据，只需要**二元关系**（预约/未预约）
- 计算简单，效率高

#### 示例

```
集合A = {1, 2, 3}
集合B = {2, 3, 4}

交集 = {2, 3} → 2个元素
并集 = {1, 2, 3, 4} → 4个元素

Jaccard相似度 = 2/4 = 0.5
```

### 其他相似度计算方法

#### 1. 余弦相似度（Cosine Similarity）

```
Cosine(A, B) = (A · B) / (||A|| × ||B||)
```

适用于有评分数据的场景。

#### 2. 皮尔逊相关系数（Pearson Correlation）

```
Pearson(A, B) = Σ((A_i - Ā)(B_i - B̄)) / √(Σ(A_i - Ā)² × Σ(B_i - B̄)²)
```

适用于评分数据，考虑用户评分习惯的差异。

#### 3. 欧几里得距离（Euclidean Distance）

```
Euclidean(A, B) = √(Σ(A_i - B_i)²)
```

距离越小，相似度越高。

### 关键方法实现

#### 1. 用户相似度计算

```java
public Double calculateUserSimilarity(Long userId1, Long userId2) {
    // 获取两个用户的预约记录
    List<SkiCourseAppointment> user1Appointments = getUserAppointments(userId1);
    List<SkiCourseAppointment> user2Appointments = getUserAppointments(userId2);
    
    // 构建教练ID集合
    Set<Long> user1Coaches = user1Appointments.stream()
        .map(SkiCourseAppointment::getCoachId)
        .collect(Collectors.toSet());
    
    Set<Long> user2Coaches = user2Appointments.stream()
        .map(SkiCourseAppointment::getCoachId)
        .collect(Collectors.toSet());
    
    // 计算交集和并集
    Set<Long> intersection = new HashSet<>(user1Coaches);
    intersection.retainAll(user2Coaches);
    
    Set<Long> union = new HashSet<>(user1Coaches);
    union.addAll(user2Coaches);
    
    // Jaccard相似度
    return union.isEmpty() ? 0.0 : (double) intersection.size() / union.size();
}
```

#### 2. 教练相似度计算

```java
public Double calculateCoachSimilarity(Long coachId1, Long coachId2) {
    // 获取两个教练的学员集合
    Set<Long> coach1Students = getCoachStudents(coachId1);
    Set<Long> coach2Students = getCoachStudents(coachId2);
    
    // 计算交集和并集
    Set<Long> intersection = new HashSet<>(coach1Students);
    intersection.retainAll(coach2Students);
    
    Set<Long> union = new HashSet<>(coach1Students);
    union.addAll(coach2Students);
    
    // Jaccard相似度
    return union.isEmpty() ? 0.0 : (double) intersection.size() / union.size();
}
```

## 算法优势与局限性

### 优势

#### 1. 无需用户输入偏好
- 基于用户行为数据自动学习
- 用户无需填写复杂的偏好表单

#### 2. 能发现潜在兴趣
- 推荐用户可能感兴趣但未发现的教练
- 帮助用户探索新的选择

#### 3. 推荐结果可解释
- 显示推荐理由（基于相似用户/相似教练）
- 用户能理解为什么推荐这个教练

#### 4. 适应性强
- 随着数据积累，推荐效果会越来越好
- 能适应用户兴趣的变化

### 局限性

#### 1. 冷启动问题（Cold Start）

**问题描述：**
- 新用户没有历史数据，无法计算相似度
- 新教练没有学员数据，无法计算相似度

**解决方案：**
- 新用户：推荐热门教练（按学员数、评分排序）
- 新教练：暂时不推荐，等待积累数据

#### 2. 数据稀疏性问题（Sparsity）

**问题描述：**
- 用户-教练矩阵非常稀疏
- 大部分用户只预约过少数教练
- 导致相似度计算不准确

**解决方案：**
- 设置相似度阈值（> 0.1）
- 使用混合推荐策略
- 结合内容过滤（Content-Based）算法

#### 3. 计算复杂度

**问题描述：**
- 用户数量大时，计算所有用户相似度耗时
- 教练数量大时，计算所有教练相似度耗时

**优化方案：**
- 限制相似用户/教练数量（前50个）
- 使用缓存机制
- 异步计算推荐结果

#### 4. 推荐多样性不足

**问题描述：**
- 容易推荐热门教练，导致推荐结果同质化
- 小众但优质的教练难以被推荐

**解决方案：**
- 引入多样性因子
- 结合内容特征（教练等级、擅长领域等）
- 使用多目标优化

## 总结

协同过滤推荐算法是一种经典的推荐算法，通过分析用户行为数据，发现用户和物品之间的相似性，从而为用户推荐可能感兴趣的物品。

在滑雪场管理系统中，我们实现了：
1. **基于用户的协同过滤**：找到相似用户，推荐他们喜欢的教练
2. **基于物品的协同过滤**：找到相似教练，推荐给用户
3. **混合推荐策略**：结合两种算法的优势，提高推荐准确性

算法使用Jaccard相似度计算相似性，适合处理二元关系数据（预约/未预约），无需评分数据即可工作。

通过合理的参数设置和优化策略，算法能够为用户提供个性化的教练推荐，提升用户体验和系统价值。
