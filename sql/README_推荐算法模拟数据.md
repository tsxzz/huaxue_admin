# 协同过滤推荐算法模拟数据使用说明

## 📋 文件说明

本目录包含以下SQL脚本文件，用于生成推荐算法的测试数据：

1. **`mock_users_for_recommendation.sql`** - 创建用户数据（教练和学员）
2. **`mock_courses_for_recommendation.sql`** - 创建课程和时间表数据
3. **`mock_data_for_recommendation.sql`** - 创建教练信息和预约数据

## 🚀 执行顺序

请按照以下顺序执行SQL脚本：

### 步骤1：创建用户数据
```sql
-- 执行 mock_users_for_recommendation.sql
-- 创建10个教练用户（user_id: 105-114）
-- 创建20个学员用户（user_id: 115-134）
```

### 步骤2：创建课程和时间表数据
```sql
-- 执行 mock_courses_for_recommendation.sql
-- 为每个教练创建1个课程（course_id: 10-19）
-- 为每个课程创建多个时间表（schedule_id: 100-129）
```

### 步骤3：创建教练信息和预约数据
```sql
-- 执行 mock_data_for_recommendation.sql
-- 创建10个教练信息记录
-- 创建40+条课程预约记录（体现协同过滤效果）
```

## 📊 数据设计说明

### 用户分组

#### 用户组A（115-119）：单板偏好组
- **用户115**：预约过教练105（单板）、108（单板）
- **用户116**：预约过教练105（单板）、109（单板）
- **用户117**：预约过教练108（单板）、109（单板）
- **用户118**：预约过教练105（单板）、111（单板）
- **用户119**：预约过教练108（单板）、111（单板）

**特点**：这些用户都喜欢单板教练，相似度高，会互相推荐单板教练。

#### 用户组B（120-124）：双板偏好组
- **用户120**：预约过教练106（双板）、107（双板）
- **用户121**：预约过教练106（双板）、110（双板）
- **用户122**：预约过教练107（双板）、110（双板）
- **用户123**：预约过教练106（双板）、112（双板）
- **用户124**：预约过教练107（双板）、112（双板）

**特点**：这些用户都喜欢双板教练，相似度高，会互相推荐双板教练。

#### 用户组C（125-129）：混合偏好组
- **用户125**：预约过教练105（单板）、106（双板）
- **用户126**：预约过教练108（单板）、107（双板）
- **用户127**：预约过教练109（单板）、110（双板）
- **用户128**：预约过教练111（单板）、112（双板）
- **用户129**：预约过教练105（单板）、113（双板）

**特点**：这些用户既有单板也有双板偏好，推荐结果会比较多样化。

#### 用户组D（130-134）：新用户组
- 每个用户只预约过1个教练
- 数据较少，用于测试冷启动场景

## 🧪 测试场景

### 场景1：用户115登录，查看推荐
**预期结果**：
- 推荐单板教练（109、111），因为相似用户116、117、118、119都喜欢这些教练
- 推荐理由："基于相似用户的推荐"

### 场景2：用户120登录，查看推荐
**预期结果**：
- 推荐双板教练（107、110、112），因为相似用户121、122、123、124都喜欢这些教练
- 推荐理由："基于相似用户的推荐"

### 场景3：用户125登录，查看推荐
**预期结果**：
- 推荐结果会比较多样化，因为该用户既有单板也有双板偏好
- 可能推荐单板教练（108、109）或双板教练（107、110）

### 场景4：用户130登录，查看推荐
**预期结果**：
- 可能推荐热门教练，因为该用户数据较少（冷启动问题）
- 推荐理由："热门教练推荐"

## 📝 注意事项

1. **角色ID**：请根据实际的`sys_role`表中的`role_id`调整`mock_users_for_recommendation.sql`中的角色分配
   - 默认：教练角色ID = 101，游客角色ID = 100

2. **密码**：所有测试用户的默认密码都是 `123456`

3. **数据清理**：如果需要重新生成数据，请先执行删除语句清理旧数据

4. **外键约束**：确保执行顺序正确，避免外键约束错误

## 🔍 验证数据

执行完所有脚本后，可以使用以下SQL验证数据：

```sql
-- 查看教练信息
SELECT * FROM `ski_coach_info` WHERE `user_id` BETWEEN 105 AND 114;

-- 查看预约记录
SELECT `student_id`, `coach_id`, COUNT(*) as appointment_count 
FROM `ski_course_appointment` 
WHERE `payment_status` = '1' 
GROUP BY `student_id`, `coach_id`
ORDER BY `student_id`;

-- 查看用户相似度（示例：用户115和116）
-- 用户115预约的教练
SELECT DISTINCT `coach_id` FROM `ski_course_appointment` WHERE `student_id` = 115 AND `payment_status` = '1';
-- 用户116预约的教练
SELECT DISTINCT `coach_id` FROM `ski_course_appointment` WHERE `student_id` = 116 AND `payment_status` = '1';
```
